## 2. 已识别薄弱点（非失败但建议优化）

## F-01 复合故障路径覆盖不足（中优先级）
- 现象：当前已覆盖单一故障注入，但“断网 + 部分成交 + 重启”复合场景尚未进入常规门禁。
- 影响：极端组合故障下仍存在行为不确定性。
- 修复建议：
  - 新增复合故障矩阵测试并纳入 CI nightly。
  - 针对复合故障输出单独恢复时间与一致性指标。

## F-02 CI 依赖环境差异风险（中优先级）
- 现象：本地使用 `.venv`，CI 使用 `pip install -e .`，解释器与依赖版本可能漂移。
- 影响：本地通过但 CI 偶发失败。
- 修复建议：
  - 固化 Python 次版本（3.11.x）并锁定依赖版本。
  - 增加 `pip freeze` 工件上传与失败时环境快照。

## F-03 Chaos 矩阵维度不足（低优先级）
- 现象：当前覆盖 5 类故障，但未覆盖“长时网络抖动 + 部分成交 + 重启组合故障”。
- 影响：复合故障行为仍有未知风险。
- 修复建议：
  - 新增参数化 Chaos 矩阵，覆盖复合故障路径。
  - 将组合故障加入 nightly 回归任务。
